# 1. Базовые общие понятия машинного обучения и искусственного интеллекта: в чём их разница? Назовите основные виды машинного обучения.

Искусственный интеллект - попытки создать компьютерные программы, врнешне подражающие поведению человеческого мышления. (цитата из лекции 1)

Машинное обучение - раздел искусственного интеллекта, занимающийся анализом данных, нахождением и извлечением из них зависимостей и закономерностей, и выработкой на их основе обоснованных решений. (цитата из лекции 1)

Основные виды МО:
- Обучение с учителем: модель учится на размеченном наборе данных.
- Обучение без учителя: модель учится на неразмеченном наборе данных.
- Обучение с подкреплением: модель учится путем взаимодействия со средой.

Не очень основные, но упомянутые в лекциях виды МО:
- Обучение с частичным привлечением учителя: модель учится на размеченных и на неразмеченных данных.
- Самостоятельное обучение.
- Состязательное обучение: две модели, решающие дополняие друг друга задачи (например задача генерации картинки и задача класификации картинки на сгенерированную/не сгенерирвоанную).

(виды взяты из лекций)

# 2. Какова общая последовательность действий при создании системы машинного обучения (решателя)? В чём назначение каждой из этих операций?

1. Определить и формализовать задачу (серва инженерно, затем математически):
    - что дано,
    - что требуется определить,
    - какими свойствами должно характеризоваться решение.
2. Выбрать модель данных/систем. Очень часто модель - это вид аппроксимации.
3. Выбрать адеватный вид представления данных. Очень часто упускается тот важнейший факт, что одна и та же информаия может быть представлена огромным числом способов. От выбора наилучшего способа зависит все:
    - точность вычисления параметров модели и даже корректность результатов,
    - вычислительная эффективность (по скорости и объему памяти),
    - наглядность результатов для человека.
4. Подготовить данные для обучения:
    - собрать,
    - очистить от шума, выбросов, ошибок, повторов, статистической неоднородности (data imbalance),
    - разделить на выборки разного назначения (например, обучающую и проверочную).
5. Инициализировать параметры модели.
6. Задать матрики качества обучения.
7. Обучить модель - это и есть найти приемленные (в обчыно недостижимом идеале - оптимальные, в очень успешном случае - субоптимальные) ее параметры.
8. Проверить качество модели и ее устойчивость к вариациям данных.

(Взято из лекций, в семинарах примерно тоже самое, но более кратко)

# 3. Машинное обучение с учителем (supervised learning): чем оно отличается от других видов машинного обучения и что специфическое для него необходимо? Какими средствами это достигается?

Задача обучения с учителем формулируется следующем образом: \
Есть набор пар объект-метка $( x^{(i)}, y^{(i)} )_{i=1}^\infty$. Объекты пренадлежат множеству $X$, а метки, соответственно, множеству меток $Y$. Необходимо аппроксимировать функцию $f: X \to Y$, сопоставляющую объект метке.

Для машинного обучения с учителем необходима база данных пар объект-метка. Такие базы даннх получаются ручной разметкой, автоматической разметкой и другими хитроумными способами.

# 4. Методы решения задач обучения с подкреплением (reinforcement learning). Опишите задачи, решаемые методами обучения с подкреплением.

Агент взаимодействет со средой. Цель - максимизировать кумулятивную награду в долгосрочной перспективе. Обучение идет методом проб и ошибок.

Q-обучение (Q-learning):
$$
Q_{i+1}(s_t, a_t) = Q_i(s_t, a_t) + \alpha [r_{t+1} + \max_{a'} \gamma Q_i(s_{i+1}, a') - Q_i(s_t, a_t)],
$$
где \
$s_t$ - состояние (state), \
$a_t$ - действие (action), \
$Q_i$ - функция качества (quality function)

Глубокое Q-обучение (deep Q-lerning) - Функция $Q_i$ аппроксимируется глубокой нейросетью.

SARSA алгоритмы (state-action-reward-state-action) - действие предпринимается согласно текущему состоянию и стратегии, а не после оптимизации по всему опыту состояний и стратегий:
$$
Q_{i+1}(s_t, a_t) = Q_i(s_t, a_t) + \alpha [r_{t+1} + \gamma Q(s_{t+1}, a_{t+1}) - Q_i(s_t, a_t)]
$$

(Q-обучение, глубокое Q-обучение и SARSA из лекций)

# 5. Как подготавливают числовые и категориальные данные для использования в системахмашинного обучения? Перечислите операции и их назначение. В чём принципиальная разница предобработки естественным образом числовых и категориальных данных?

Общий конвейер предобработки данных:
1. Сбор и объединение данных.
2. Исследовательский анализ данных (EDA): понимание распределений, поиск аномалий и зависимостей.
3. Обработка пропущенных значений.
4. Обработка выбросов (для числовых данных).
5. Нормализация/стандартизация (для числовых данных).
6. Кодирование категориальных признаков.
7. Понижение размерности (опционально).
8. Разделение на обучающую и тестовую выборки (важно делать после всей предобработки, но с использованием параметров, полученных только на обучающей выборке).

Предобработка числовых данных:
- обработка пропусков
    - удаление
    - замена статистиками (средним, меданой, модой)
- обработка выбрасов:
    - IQR
    - z-score
- масштабирвоание/нормализация:
    - стандартизация
    - ман-макс норнмализация
    - robust scaling
- преобразование распределния
    - логарифмирование

Предобработка категориальных данных:
- обработка пропусков:
    - замена на моду
    - преобразование пропусков в отдельную категорию
- кодирование:
    - label encoding (0, 1, 2, 3, ...), применимо только для порядковых (ординальных) данных
    - one-hot
    - target encoding
    - frequency encoding

Для числовых данных мы работаем с их распределением и масштабом. Мы заботимся о том, чтобы все числа были в одной "системе координат" и не было точек, ломающих эту систему. Для категориальных данных мы работаем с их смыслом и взаимосвязью с целью. Наша задача — перевести "слова" (категории) в "цифры" так, чтобы модель не исказила их смысл и смогла извлечь полезные закономерности.

# 6. К какому типу машинного обучения относится регрессия? Что такое регрессор? Какими свойствами хороший регрессор должен быть наделён? По каким параметрам судят о качестве регрессионной модели?

К обучению с учителем. Регрессор - модель, предсказывающее целевое значение по объекту. Хороший регрессор должен обладать высокой точностью на ранее не виденных данных. О качестве регрессора судят по метрикам на ранее не виденных данных: (R)MAE, (R)MSE, $R^2$.
